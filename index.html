AES-GCM · WebCrypto

Symmetric Encryption

One key encrypts and decrypts. AES-GCM bundles in authentication, so tampering with the ciphertext is detected automatically.

Keys and plaintext never leave this browser tab.
How this works
A random 256-bit key is generated if you don't supply one. Each encryption uses a fresh 12-byte IV, prepended to the ciphertext so decryption can recover it. Reusing an IV with the same key breaks AES-GCM's security guarantees — this tool always generates a new one per operation.
RSA-OAEP · WebCrypto

Asymmetric Encryption

Encrypt with a public key, decrypt only with the matching private key. Generate a pair on the Key Generation page first.

How this works
RSA-OAEP is limited to small payloads (well under the key size) — it's meant for wrapping a symmetric key, not encrypting large documents directly. For bulk data, encrypt with AES-GCM and use RSA only to protect the AES key.
Local files

File Tools

Hash a file or encrypt it with AES-GCM. The file is read and processed in memory — it's never uploaded anywhere.

Digests

Hashing

Turn data into a fixed-size fingerprint. Use SHA-256 or SHA-512 for integrity checks; avoid MD5 and SHA-1 outside legacy compatibility.

Message authentication

HMAC

Proves both integrity and authenticity — only someone holding the key could have produced this tag.

RSA-PSS · WebCrypto

Digital Signature

Sign with a private key; anyone with the public key can verify the signature came from you and the message wasn't altered.

Verification

Compare Values

Paste two hashes, HMACs, or signatures to confirm they match exactly — a manual step people skip and shouldn't.

Key material

Key Generation

Generate real, exportable keys for the tools above — AES for symmetric work, RSA for encryption and signatures.

Key derivation

PBKDF2 & Argon2id

Turn a password into a key or storable hash, slowly and deliberately, so brute-forcing it is expensive. Argon2id is the current recommended default; PBKDF2 remains widely supported.

ECDH · WebCrypto

Key Exchange

Simulates two parties (A and B) deriving the same shared secret over an insecure channel using elliptic-curve Diffie–Hellman — each generates a keypair, exchanges public keys, and independently arrives at an identical secret.

Tokens

JWT Tool

Decode, verify, or issue JSON Web Tokens signed with HMAC (HS256) or RSA (RS256).

Transport formats

Encoding

Convert text to and from Base64 or hexadecimal — encoding, not encryption; anyone can reverse it.

Passwords

Password & Entropy

Generate a strong random password, or check how much entropy an existing string carries.
