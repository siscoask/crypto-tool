Symmetric Encryption

A powerful AES-GCM encryption tool for secure data protection with integrated authentication.

Symmetric Encryption (AES-GCM)

AES-GCM

What is Symmetric Encryption?

Symmetric encryption uses a single key for both encryption and decryption. AES-GCM combines encryption with authentication for robust security.

How to Use: Enter text to encrypt. Optionally provide a base64-encoded key (leave blank to generate one). For decryption, provide the encrypted data and the same key.

Common Errors:
- "Invalid key": Ensure the key is valid base64 and 32 bytes long.
- "Invalid encrypted data": Check that the input is valid base64 with a 12-byte IV.
- "Empty input": Provide text to encrypt or decrypt.

Asymmetric Encryption

Secure RSA-OAEP encryption for key exchange and small data encryption using public/private key pairs.

Asymmetric Encryption (RSA-OAEP)

RSA-OAEP

What is Asymmetric Encryption?

Uses a public/private key pair for secure encryption. Encrypt with the public key and decrypt with the private key.

How to Use: Generate keys in the "Key Generation" tab. Enter text and public key for encryption, or encrypted data and private key for decryption.

Common Errors:
- "Invalid key": Ensure keys are valid PEM format.
- "Data too large": RSA is limited to small data.
- "Invalid encrypted data": Check base64 validity.

Hashing

Generate fixed-size digests for data integrity and password storage with multiple algorithms.

Hashing Tools

What is Hashing?

Creates a fixed-size digest from data for integrity checks or password storage. Supports SHA-1, SHA-256, SHA-512, MD5, and Blake2b.

How to Use: Enter text, select an algorithm, and click "Hash".

Common Errors:
- "Empty input": Provide text to hash.

HMAC

Generate Hash-based Message Authentication Codes for data integrity and authenticity.

HMAC Generation

What is HMAC?

Ensures data integrity and authenticity using a key. Supports HMAC with SHA-256 and SHA-512.

How to Use: Enter text and a key, select an algorithm, and click "Generate HMAC".

Common Errors:
- "Empty input or key": Provide both text and a key.
- "Invalid key": Ensure the key is valid text.

PBKDF2

Derive secure keys from passwords using PBKDF2 for storage or key generation.

Key Derivation

What is PBKDF2?

Derives a secure key from a password and salt, ideal for secure storage or key generation.

How to Use: Enter a password and optional salt (base64-encoded). Click "Derive Key".

Common Errors:
- "Empty password": Provide a password.
- "Invalid salt": Ensure the salt is valid base64 or leave blank.

Digital Signature

Ensure data authenticity and integrity with RSA-PSS digital signatures.

Signature Tools

RSA-PSS

What is a Digital Signature?

Ensures data authenticity using a private key to sign and a public key to verify.

How to Use: Generate keys in the "Key Generation" tab. Enter text and private key to sign, or text, signature, and public key to verify.

Common Errors:
- "Invalid key": Ensure keys are valid PEM format.
- "Empty input": Provide text and appropriate keys.
- "Invalid signature": Check signature validity.

Utilities

Tools for generating random passwords and analyzing data entropy.

Utility Tools

What are Utilities?

Includes random password generation and entropy analysis for data randomness.

How to Use: Specify password length and click "Generate Password", or enter text and click "Analyze Entropy".
